# Пользовательское руководство

## Назначение руководства

Руководство описывает работу пользователей в системе автоматизации пункта выдачи заказов. Документ помогает выполнить основные действия: принять товар на склад, создать заказ, провести его через сборку, выдать клиенту и проверить результат в отчетах.

## Роли пользователей

- Сотрудник ПВЗ работает с заказами, складскими остатками, приемкой товара и выдачей заказов клиентам.
- Управляющий выполняет операции сотрудника, просматривает отчетность и управляет пользователями и товарами.
- Руководитель просматривает отчетность и контролирует показатели работы пункта выдачи.
- Системный администратор управляет учетными записями и справочником товаров.

## Вход в систему

1. Откройте приложение в браузере.
2. Нажмите «Войти».
3. Введите логин и пароль.
4. После входа используйте меню в верхней части страницы для перехода к доступным разделам.

Если нужный раздел не отображается, проверьте, что вход выполнен под пользователем с подходящей ролью.

## Просмотр складских остатков

Откройте раздел «Склад». На странице отображается список товаров и текущее количество каждого товара на складе. Эти данные используются при создании заказов и приемке товара.

## Приемка товара

1. Перейдите в раздел «Склад».
2. Выберите товар из списка.
3. Укажите количество, которое поступило на склад.
4. Подтвердите приемку.

После приемки количество товара на складе увеличивается. В журнале складских операций появляется запись о приеме товара.

## Создание заказа

1. Перейдите в раздел «Заказы».
2. Нажмите «Создать заказ».
3. Укажите номер заказа.
4. Выберите товар.
5. Укажите количество.
6. Нажмите «Создать».

Система проверяет доступный остаток. Если товара достаточно, заказ появляется в списке заказов.

## Перевод заказа в сборку

В списке заказов найдите нужный заказ и нажмите «В сборку». После этого заказ считается переданным на подготовку.

## Подтверждение комплектации

После фактической подготовки товара нажмите «Подтвердить комплектацию» у заказа в сборке. Система уменьшит складской остаток и сохранит операцию списания в журнале.

После подтверждения комплектации заказ готов к выдаче клиенту.

## Выдача заказа клиенту

1. В списке заказов найдите заказ, готовый к выдаче.
2. Нажмите «Выдать».
3. Введите документ получателя, например паспорт или другой идентификатор.
4. Подтвердите выдачу.

После подтверждения заказ считается выданным. Документ получателя сохраняется в карточке заказа, а в журнале складских операций появляется запись о выдаче.

## Просмотр журнала складских операций

Откройте журнал операций в складском разделе. В журнале отображаются приемы, списания и выдачи товаров. Для каждой записи показаны дата, товар, тип операции, количество, оператор и комментарий.

Журнал помогает проверить, почему изменился складской остаток, какие действия выполнялись с товаром и кто их выполнил.

## Просмотр отчетности

Раздел «Отчеты» доступен пользователям с правами просмотра отчетов. В нем отображаются общие показатели по заказам, количество выданных заказов, количество заказов, готовых к выдаче, последние складские операции и отчет по выданным заказам с датой выдачи.

## Экспорт выданных заказов в CSV

В разделе «Отчеты» откройте отчет по выданным заказам и выберите экспорт в CSV. Файл содержит номер заказа, дату создания, дату выдачи, документ получателя и состав заказа. Его можно использовать для передачи данных, проверки на защите или дальнейшей обработки в табличном редакторе.

## Типовые ошибки

### Недостаточно товара на складе

Если при создании заказа указано количество больше доступного остатка, система не создаст заказ и покажет сообщение с доступным количеством. Уменьшите количество в заказе или сначала выполните приемку товара.

### Заказ нельзя выдать без документа получателя

При выдаче заказа нужно указать документ получателя. Если поле оставить пустым, система не завершит выдачу и попросит заполнить документ.

### Действие недоступно для роли

Некоторые разделы и действия доступны только определенным ролям. Например, отчетность доступна управляющему и руководителю, а администрирование пользователей и товаров - управляющему и системному администратору. Если действие недоступно, войдите под пользователем с нужной ролью или обратитесь к администратору.

## Основной рабочий сценарий

1. Выполнить приемку товара на склад.
2. Создать заказ на доступное количество товара.
3. Перевести заказ в сборку.
4. Подтвердить комплектацию.
5. Выдать заказ клиенту с указанием документа получателя.
6. Проверить складские операции и отчет по выданным заказам.
