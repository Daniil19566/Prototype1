# Отчет о ручном функциональном тестировании

## Цель тестирования

Цель тестирования - проверить основные пользовательские сценарии системы автоматизации пункта выдачи заказов: работу с ролями, складскими остатками, приемкой товара, заказами, выдачей, журналом операций, отчетностью и администрированием справочников.

## Окружение

- .NET 8
- ASP.NET Core MVC
- PostgreSQL
- Браузер

## Проверенные роли

- `employee`
- `manager`
- `lead`
- `sysadmin`

## Проверенные сценарии

- вход в систему;
- просмотр складских остатков;
- приемка товара;
- создание заказа;
- запрет создания заказа при нехватке остатка;
- перевод заказа в сборку;
- подтверждение комплектации;
- выдача заказа;
- запрет выдачи без документа получателя;
- просмотр журнала операций;
- просмотр отчетности;
- экспорт выданных заказов в CSV;
- создание пользователя;
- удаление пользователя;
- создание товара;
- удаление свободного товара;
- запрет удаления связанного товара.

## Автоматическое тестирование

Для базовой автоматической проверки используется xUnit. Тестовый проект: `Prototype.Tests`.

Покрытые автотестами проверки:

- display-helper'ы статусов заказов;
- display-helper'ы типов складских операций;
- жизненный цикл заказа от создания до выдачи;
- запрет создания заказа при нехватке товара;
- запрет выдачи заказа без документа получателя;
- корректное CSV-экранирование в отчете выданных заказов;
- сохранение оператора для приемки товара, списания при комплектации и выдачи заказа.

Команда запуска:

```bash
dotnet test
```

Результат выполнения:

- `Passed: 12`
- `Failed: 0`

## Итоговый вывод

Ручное функциональное тестирование и базовые автоматические тесты подтвердили, что основные функции системы работают: пользователи входят в систему, складские остатки обновляются, заказы проходят полный путь от создания до выдачи, операции фиксируются в журнале, отчеты доступны пользователям с нужными правами, а административные действия выполняются с необходимыми ограничениями.

Критичные сценарии склада, заказов, отчетности и администрирования закрыты ручными проверками. Автоматические тесты покрывают отображение пользовательских названий, основной жизненный цикл заказа, важные пользовательские ошибки при создании и выдаче заказа, CSV-экранирование в отчете выданных заказов и сохранение оператора складских операций. Оставшийся риск: набор автотестов пока базовый, поэтому дополнительные регрессионные проверки бизнес-сценариев следует расширять по мере развития системы.
