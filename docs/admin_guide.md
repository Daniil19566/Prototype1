# Руководство администратора

## Назначение руководства

Руководство описывает административные и эксплуатационные действия в системе автоматизации пункта выдачи заказов. Документ предназначен для пользователей, которые отвечают за учетные записи, справочник товаров, контроль отчетности и базовую эксплуатацию системы.

## Кто использует

- Управляющий контролирует работу пункта выдачи, пользователей, товары и отчетность.
- Системный администратор управляет учетными записями, правами доступа и справочником товаров.

## Стартовые учетные записи

| Логин | Пароль | Назначение |
| --- | --- | --- |
| `manager` | `12345` | Управляющий |
| `sysadmin` | `12345` | Системный администратор |

После внедрения системы стандартные пароли необходимо заменить.

## Доступ к разделу администрирования

1. Войдите в систему под учетной записью управляющего или системного администратора.
2. В верхнем меню откройте раздел «Администрирование».
3. Если раздел не отображается, проверьте роль текущего пользователя.

## Управление пользователями

### Создание пользователя

1. Откройте раздел «Администрирование».
2. В блоке управления пользователями укажите логин.
3. Укажите временный пароль.
4. Выберите роль пользователя.
5. Нажмите «Создать пользователя».

После создания пользователь появляется в списке и может войти в систему под указанным логином и паролем.

### Выбор роли

При создании пользователя выберите роль по его обязанностям:

- сотрудник работает с заказами, складом и выдачей;
- управляющий работает с заказами, складом, отчетами и администрированием;
- руководитель просматривает отчетность;
- системный администратор управляет пользователями и товарами.

Не выдавайте расширенные права пользователям, которым они не нужны для работы.

### Удаление пользователя

В списке пользователей нажмите «Удалить» рядом с нужной учетной записью и подтвердите действие.

Ограничения удаления:

- нельзя удалить самого себя;
- нельзя удалить последнего пользователя с административной ролью.

Эти ограничения защищают систему от ситуации, когда в ней не остается пользователя, способного управлять доступом.

## Управление товарами

### Создание товара

1. Откройте раздел «Администрирование».
2. В блоке справочника товаров укажите наименование товара.
3. При необходимости заполните описание.
4. Нажмите «Добавить товар».

Новый товар появится в справочнике. После этого его можно использовать при приемке и создании заказов.

### Удаление свободного товара

Товар можно удалить, если он еще не использовался в заказах и по нему нет складских операций. В списке товаров нажмите «Удалить» и подтвердите действие.

### Ограничения удаления товара

Нельзя удалить товар в следующих случаях:

- товар используется в заказах;
- по товару есть складские операции.

Если товар уже участвовал в рабочих процессах, его удаление может нарушить историю заказов, остатков и отчетности. В таком случае товар следует оставить в справочнике.

## Контроль отчетности

Раздел «Отчеты» используется для контроля работы пункта выдачи.

Администратор или управляющий может:

- просмотреть общее количество заказов;
- просмотреть количество выданных заказов;
- просмотреть количество заказов, готовых к выдаче;
- проверить последние складские операции и оператора, который выполнил действие;
- открыть отчет по выданным заказам с датой выдачи;
- экспортировать выданные заказы в CSV.

CSV-файл можно использовать для передачи данных, архивирования или проверки результатов демонстрационного сценария. В выгрузке есть дата создания заказа и дата его выдачи.

## Рекомендации по эксплуатации

- Регулярно проверяйте складские остатки, особенно перед созданием крупных заказов.
- Проверяйте журнал операций после приемки, списания и выдачи товара. В журнале видно, кто выполнил каждую операцию.
- Не используйте одну учетную запись для всех сотрудников: действия должны быть связаны с конкретным пользователем.
- Меняйте стандартные пароли после внедрения системы.
- Создавайте пользователей только с теми ролями, которые нужны им для работы.
- Перед удалением товара убедитесь, что он не нужен для анализа истории заказов и складских операций.
